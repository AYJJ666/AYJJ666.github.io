C++多线程项目笔记
Created 2020-07-12| Updated 2020-07-12
Post View:

根据学习路线记录学习的知识内容以及理解，发现问题请联系作者。

一.进程、线程与并发

  • 进程

    • 指在系统中正在运行的一个应用程序
    • 程序一旦运行就是进程
    • 进程——资源分配的最小单位

  • 线程

    • 是操作系统能够进行运算调度的最小单位。它被包含在进程之中，是进程中的实际运作单位
    • 一条线程指的是进程中一个单一顺序的控制流

  • 进程与线程的关系

    • 线程是程序执行的最小单位，而进程是操作系统分配资源的最小单位
    • 一个进程中可以并发多个线程，每条线程并行执行不同的任务（代码段）
    • 进程之间相互独立，但同一进程下的各个线程之间共享程序的内存空间(包括代码段，数据集，堆等)及一些进程级的资源(如打开文件和信号等)，某进程内的线程在其他进程不可见
    • 调度和切换：线程上下文切换比进程上下文切换要快得多

  • 并发

    • 两个或更多地任务（独立活动，进程）同时发生（进行）
    • 一个程序同时执行多个独立的任务

  • 关于

    • 主线程
      • 一个进程至少包含一个线程，即主线程，主线程是惟一的
      • 主线程与进程拥有共同的生命周期，即进程诞生主线程诞生，进程结束主线程结束
    • 其他线程（子线程）
      • 主动按需创建
      • 有独立的堆栈空间，有自己的生命周期
      • 合理使用线程可以提高程序运行效率，但线程之间的切换需要消耗额外的时间、内存等资源，所以要根据需求创建，并不是越多越好

二.并发的实现

1. 使用thread实现

  • 包含头文件thread

  • 使用的函数

    • thread name(fun*){}
      • name为线程对象名，fun为已有函数
      • 无返回值
      • 需要传入线程中执行的函数的指针（fun*)，线程创建后开始执行fun函数，fun函数执行结束则线程执行结束
    • name.join()
      • join()函数的作用是等待调用本函数的线程结束
      • 如：在线程A中调用了线程B的B.join()，则A线程执行到B.join();时会停止并等待B线程执行，B线程执行结束后，A线程继续执行。

  • 示例

lock_guard

概述
根据对象的析构函数自动调用的原理，c++11推出了std::lock_guard自动释放锁，其原理是：声明一个局部的lock_guard对象，在其构造函数中进行加锁，在其析构函数中进行解锁。最终的结果就是：在定义该局部对象的时候加锁（调用构造函数），出了该对象作用域的时候解锁（调用析构函数）。

使用方法
1.首先需要包含mutex头文件
2.然后创建一个锁 std::mutex mutex
3.在需要被加锁的作用域内 将mutex传入到创建的std::lock_guard局部对象中
